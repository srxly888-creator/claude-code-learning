## 📝 **贡献流程**

### **Step 1: Fork 仓库**
```bash
# 在 GitHub 上 Fork 这个仓库
```

### **Step 2: 克隆到本地**
```bash
git clone https://github.com/YOUR_USERNAME/claude-code-learning.git
cd claude-code-learning
```

### **Step 3: 创建分支**
```bash
git checkout -b feature/your-feature-name
```

### **Step 4: 做出修改**
```bash
# 编辑文件
# 添加新文件
# 改进现有内容
```

### **Step 5: 提交更改**
```bash
git add .
git commit -m "🎯 简短描述你的修改"
```

### **Step 6: 推送到 GitHub**
```bash
git push origin feature/your-feature-name
```

### **Step 7: 创建 Pull Request**
- 在 GitHub 上创建 PR
- 描述你的修改
- 等待审核

## 📋 **内容规范**

### **文件命名**
- 使用小写字母和连字符
- 例如：`learning-guide.md`
- 避免空格和特殊字符

### **Markdown 格式**
```markdown
# 主标题

简要描述

## 二级标题

### 三级标题

- 列表项1
- 列表项2

**粗体** *斜体* `代码`

\`\`\`python
# 代码块
print("Hello")
\`\`\`
```

### **代码示例**
- 提供可运行的代码
- 添加必要注释
- 说明运行环境

### **链接引用**
```markdown
[链接文字](文件名.md)
```

## ✅ **质量标准**

### **内容质量**
- ✅ 准确性：信息正确无误
- ✅ 完整性：内容全面完整
- ✅ 实用性：可直接应用
- ✅ 可读性：易于理解

### **代码质量**
- ✅ 可运行：代码能正常执行
- ✅ 有注释：关键部分有说明
- ✅ 有测试：包含测试用例
- ✅ 遵循规范：符合编码标准

### **文档质量**
- ✅ 结构清晰：层次分明
- ✅ 语言简洁：表达清晰
- ✅ 格式统一：符合规范
- ✅ 链接有效：无死链

## 🔍 **审核流程**

### **自动检查**
- Markdown 语法检查
- 链接有效性检查
- 文件命名规范检查

### **人工审核**
1. 内容准确性
2. 格式规范性
3. 价值贡献度
4. 社区适用性

### **审核时间**
- 小修改：1-3天
- 新内容：3-7天
- 大改动：7-14天
