## 核心概念：理解 AI 法律助手

### 🤖 什么是 Claude Code

Claude Code 是 Anthropic 公司推出的命令行工具，让你能通过简单的命令调用 Claude AI 的强大能力。对于法务人员来说，它就像一个：

- 📚 **智能法律助手**: 24/7 在线的法律顾问
- 🔍 **超级搜索引擎**: 精准的法律文献检索器
- ✍️ **文档生成器**: 自动生成法律文书
- 🧠 **分析引擎**: 深度分析合同和法律文件

### 🎓 核心概念解释

#### **1. 提示词 (Prompt)**
提示词就是你给 Claude 的指令。就像给律师助理布置任务一样，提示词越清晰，结果越准确。

**简单理解**：
```
提示词 = 你的需求描述 + 背景信息 + 输出要求
```

**示例**：
```
❌ 差的提示词：
"审查这份合同"

✅ 好的提示词：
"请审查附件中的软件外包开发合同，重点关注：
1. 知识产权归属条款（第5条）
2. 违约金比例是否合理（不超过30%）
3. 是否符合《个人信息保护法》要求
请输出风险等级（高/中/低）和具体修改建议"
```

#### **2. 上下文 (Context)**
上下文是 Claude 理解你需求的背景信息。包括：
- 📄 文件内容（合同、法规、案例等）
- 🏢 业务背景（行业、公司情况等）
- 🎯 任务目标（审查、起草、分析等）

**如何提供上下文**：
- 直接上传文件
- 复制粘贴文本
- 描述背景信息

#### **3. 输出格式 (Output Format)**
你可以指定 Claude 输出的格式，方便后续使用。

**常见输出格式**：
- 📝 **Markdown**: 结构化文档
- 📊 **JSON**: 程序可读的结构化数据
- 📋 **表格**: 清晰的对比和列表
- 📄 **Word**: 直接生成可编辑文档

#### **4. 模型 (Model)**
Claude 有不同的模型版本，类似于不同级别的律师：

| 模型 | 能力 | 适用场景 | 类比 |
|------|------|---------|------|
| **Claude 3 Opus** | 最强推理能力 | 复杂法律分析、案例研究 | 资深合伙人 |
| **Claude 3 Sonnet** | 平衡性能与成本 | 日常合同审查、文档生成 | 中级律师 |
| **Claude 3 Haiku** | 快速响应 | 简单问答、格式转换 | 初级律师 |

# 保密协议（NDA）审查报告

## 📊 审查摘要

- **审查时间**: 2026-03-23 10:30
- **风险等级**: 🟡 中等
- **关键问题**: 3 个高风险，2 个中风险

---

## 🔴 高风险条款

### 1. 保密范围过于宽泛（第 2 条）

**原文**：
> "保密信息包括接收方在合作期间获知的所有信息"

**风险分析**：
- ⚠️ 范围定义模糊，可能包括公开信息
- ⚠️ 缺少"例外情形"条款
- ⚠️ 可能导致不必要的保密义务

**建议修改**：
```
保密信息是指披露方以书面、口头或其他方式提供给接收方的，
标记为"保密"或根据商业惯例应被视为保密的信息，但不包括：
(a) 已经公开的信息；
(b) 接收方在披露前已经合法获知的信息；
(c) 接收方从第三方合法获知的信息；
(d) 接收方独立开发的信息。
```

### 2. 保密期限过长（第 5 条）

**原文**：
> "保密义务在本协议终止后永久有效"

**风险分析**：
- ⚠️ 永久保密义务过于严苛
- ⚠️ 可能不符合商业合理性
- ⚠️ 建议设定合理期限

**建议修改**：
```
保密义务在本协议终止后继续有效，期限为【3】年。
但对于商业秘密等需要长期保密的信息，保密义务应持续至该信息公开为止。
```

### 3. 违约责任不对等（第 8 条）

**原文**：
> "接收方违反保密义务需支付违约金 100 万元"

**风险分析**：
- ⚠️ 违约金金额过高，可能被法院调整
- ⚠️ 仅约定接收方责任，缺少披露方责任
- ⚠️ 缺少损失赔偿计算方式

**建议修改**：
```
任何一方违反本协议约定的保密义务，应向守约方支付违约金人民币【20】万元；
如违约金不足以弥补守约方实际损失的，违约方还应赔偿差额部分。
```

---

## 🟡 中风险条款

### 4. 争议解决条款不明确（第 10 条）

**原文**：
> "争议提交人民法院诉讼解决"

**风险分析**：
- ⚠️ 未明确管辖法院
- ⚠️ 可能导致异地诉讼
- ⚠️ 建议明确管辖法院或选择仲裁

**建议修改**：
```
因本协议引起的或与本协议有关的任何争议，双方应首先通过友好协商解决；
协商不成的，任何一方均可向【原告所在地】人民法院提起诉讼。
```

### 5. 缺少不可抗力条款

**风险分析**：
- ⚠️ 未约定不可抗力情形下的责任免除
- ⚠️ 建议补充不可抗力条款

**建议补充**：
```
因不可抗力（包括但不限于自然灾害、战争、政府行为、疫情等）导致本协议
无法履行或延迟履行的，受影响方不承担违约责任，但应及时通知对方并提供
相关证明。
```

创建文件 `contract-review-prompt.md`：

```markdown
# 合同审查任务

## 基本信息
- **合同名称**: 采购合同
- **合同双方**: 甲方（我方，采购方） vs 乙方（ABC公司，供应方）
- **合同金额**: ¥500,000
- **合同标的**: IT 设备采购
- **行业**: 科技

## 审查重点

### 1. 风险识别
请重点审查以下类型的风险：
- **财务风险**: 付款条件、违约金、赔偿金额
- **法律风险**: 知识产权、保密义务、争议解决
- **运营风险**: 交付标准、验收流程、质保期限

### 2. 合规检查
请检查合同是否符合以下法规：
- 《中华人民共和国民法典》
- 《中华人民共和国合同法》
- 《中华人民共和国产品质量法》（如适用）
- 《中华人民共和国消费者权益保护法》（如适用）

### 3. 关键条款
请提取并分析以下条款：
- 付款条款（第 X 条）
- 交付条款（第 X 条）
- 质量条款（第 X 条）
- 违约责任（第 X 条）
- 争议解决（第 X 条）

## 输出要求

请以 Markdown 格式输出审查报告，包含：
1. **执行摘要**: 风险等级、主要问题
2. **风险条款列表**: 按风险等级（高/中/低）分类
3. **合规检查结果**: 合规/不合规，具体问题
4. **关键条款分析**: 条款内容、风险点、建议
5. **修改建议**: 优先级排序的修改清单
6. **谈判策略**: 对我方有利的修改建议

## 特殊要求
- 我方为甲方，请站在甲方角度审查
- 关注付款条件的合理性
- 检查是否存在霸王条款
- 确保质保期限不少于 12 个月
```

#### **Q5: AI 能处理扫描件 PDF 吗？**

**A**: Claude Code 本身不直接处理 PDF，需要先提取文本。

**解决方案**：
```bash
# 方式 1: 使用 OCR 工具
brew install tesseract  # Mac
sudo apt install tesseract-ocr  # Linux

# 提取文本
tesseract contract.pdf contract.txt

# 使用 Claude Code 审查
claude-code analyze \
  --type "contract-review" \
  --file "contract.txt" \
  --output "审查报告.md"

# 方式 2: 使用 Python OCR
pip install pytesseract pdf2image

python << 'EOF'
import pytesseract
from pdf2image import convert_from_path

# 转换 PDF 为图片
pages = convert_from_path('contract.pdf')

# OCR 识别
text = ""
for page in pages:
    text += pytesseract.image_to_string(page, lang='chi_sim')

# 保存文本
with open('contract.txt', 'w', encoding='utf-8') as f:
    f.write(text)
EOF
```
