### 心得2: 逐步增加复杂度
**学习日期**: 2026-03-02
**关键收获**: 每次只增加一个功能，不要一次性添加太多。

**错误示范**:
```python
# 一次性添加所有功能 - 太复杂！
def mega_app():
    # 登录 + 数据库 + API + UI + ...
    pass
```

**正确做法**:
```python
# 第一步：只做登录
def login():
    pass

# 第二步：添加数据库
def login_with_db():
    pass
```

**感悟**: 小步快跑，逐步迭代。

---

### 心得3: 善用注释
**学习日期**: 2026-03-03
**关键收获**: 代码是写给未来的自己看的，注释是必要的。

**代码**:
```python
# ❌ 没有注释
def calc(a, b):
    return a * b * 0.1

# ✅ 有注释
def calculate_discount(price, quantity):
    """计算折扣价格，10%折扣"""
    return price * quantity * 0.1
```

**感悟**: 好的注释比好的代码更重要。
