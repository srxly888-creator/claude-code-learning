## ⚖️ 场景概述

法务工作的核心挑战：
1. **合同审查效率低**: 人工逐条审查耗时，风险识别依赖经验
2. **法律研究成本高**: 案例检索、法规解读需要大量时间和专业知识
3. **文档生成重复性**: 合同模板、法律意见书等文档生成工作量大
4. **合规风险难管控**: 法规更新频繁，合规检查容易遗漏

## 🎯 Claude Code 应用点

### **1. 合同审查**

#### **风险识别**
```bash
# 自动识别合同风险条款
claude-code analyze \
  --type "contract-risk" \
  --file "contract.pdf" \
  --jurisdiction "中国" \
  --industry "科技" \
  --output "risk-report.md"
```

**输出示例**:
```markdown
## 合同风险审查报告

### 🔴 高风险条款

#### 1. 违约金条款（第 8.2 条）
**原文**: "任何一方违约需支付合同总额 50% 的违约金"

**风险分析**:
- ⚠️ 违约金比例过高，可能被法院认定为"过分高于实际损失"
- ⚠️ 根据《合同法》第114条，违约金超过实际损失30%可请求调整
- ⚠️ 建议修改为 20%-30% 或明确计算方式

**建议修改**:
```
任何一方违约需支付合同总额 20% 的违约金，
或相当于守约方实际损失的金额（以较高者为准）
```

#### 2. 知识产权条款（第 12 条）
**原文**: "乙方在合作期间产生的所有知识产权归甲方所有"

**风险分析**:
- ⚠️ 条款过于宽泛，可能包括乙方原有知识产权
- ⚠️ 缺少"除外"条款，保护乙方原有技术
- ⚠️ 建议明确界定"合作产生"的范围

**建议修改**:
```
乙方在履行本合同过程中新产生的知识产权归甲方所有，
但乙方原有的知识产权及非因本合同产生的知识产权除外
```

### 🟡 中风险条款

#### 3. 争议解决（第 15 条）
**原文**: "争议提交甲方所在地人民法院诉讼解决"

**风险分析**:
- ⚠️ 管辖法院对乙方不利（异地诉讼成本高）
- ⚠️ 建议改为"原告所在地"或"合同履行地"

### ✅ 合规建议

1. **数据保护**: 添加个人信息保护条款（符合《个人信息保护法》）
2. **不可抗力**: 完善不可抗力条款，明确疫情等情形
3. **保密义务**: 细化保密范围和期限
```

```markdown
## 软件著作权侵权赔偿案例研究

### 📊 案例统计（2019-2024）

#### 数据概览
- **案例总数**: 1,247 件
- **原告胜诉率**: 87.3%
- **平均赔偿额**: ¥186,500
- **最高赔偿额**: ¥5,000,000
- **最低赔偿额**: ¥10,000

#### 赔偿标准分析

##### 1️⃣ 实际损失计算（占比 42%）
**适用情形**: 原告能证明实际损失
**计算方式**:

- 许可费损失
- 市场份额损失

**典型案例**:
```
案例：(2022)最高法民终123号
案情：A公司盗版B公司ERP软件，销售100套

```

##### 2️⃣ 侵权获利计算（占比 28%）
**适用情形**: 原告损失难以计算，但能证明被告获利
**计算方式**:
- 侵权产品销售额
- 侵权软件许可费
- 相关业务收入

**典型案例**:
```
案例：(2021)京民终456号
案情：C公司在其产品中嵌入盗版软件，销售获利500万
赔偿：¥1,800,000（侵权获利 × 软件贡献度）
```

##### 3️⃣ 法定赔偿（占比 30%）
**适用情形**: 实际损失和侵权获利均无法计算
**赔偿范围**: ¥500 - ¥5,000,000
**考量因素**:
- 软件类型和商业价值
- 侵权性质和情节
- 侵权持续时间
- 侵权规模和影响

**典型案例**:
```
案例：(2023)沪民终789号
案情：D公司网站使用盗版软件，侵权6个月
赔偿：¥150,000（法定赔偿）
```

### 💡 实务建议

#### 赔偿金额预估
根据案例分析，建议按以下标准预估：

1. **企业级软件侵权**: ¥500,000 - ¥2,000,000
2. **消费级软件侵权**: ¥50,000 - ¥200,000
3. **开源软件违规使用**: ¥100,000 - ¥500,000

#### 证据准备
- ✅ 软件著作权登记证书
- ✅ 软件源代码和文档
- ✅ 侵权证据（公证、截图、购买记录）
- ✅ 损失或获利证据（财务报表、销售数据）
```

### **核心代码示例**

#### **1. 合同审查核心**
```python
# src/reviewer.py

from claude_code import Client
import yaml
from typing import Dict, List
import json

class ContractReviewer:
    """合同审查助手"""
    
    def __init__(self, config_path="config/risk_rules.yaml"):
        with open(config_path, 'r', encoding='utf-8') as f:
            self.risk_rules = yaml.safe_load(f)
        self.client = Client()
    
    def review_contract(
        self,
        contract_file: str,
        review_type: str = "comprehensive",
        jurisdiction: str = "中国",
        industry: str = None
    ) -> Dict:
        """
        审查合同
        
        Args:
            contract_file: 合同文件路径
            review_type: 审查类型（comprehensive/risk/compliance）
            jurisdiction: 司法管辖区
            industry: 行业类型
        
        Returns:
            审查报告字典
        """
        # 1. 读取合同内容
        with open(contract_file, 'r', encoding='utf-8') as f:
            contract_text = f.read()
        
        # 2. 构建审查提示词
        prompt = self._build_review_prompt(
            contract_text,
            review_type,
            jurisdiction,
            industry
        )
        
        # 3. 调用 Claude Code 进行审查
        result = self.client.generate(
            prompt=prompt,
            model="claude-3-opus-20240229",
            max_tokens=8000,
            temperature=0.3  # 低温度确保准确性
        )
        
        # 4. 解析审查结果
        review_result = self._parse_review_result(result.content)
        
        # 5. 生成报告
        report = self._generate_report(review_result)
        
        return report
    
    def _build_review_prompt(
        self,
        contract_text: str,
        review_type: str,
        jurisdiction: str,
        industry: str
    ) -> str:
        """构建审查提示词"""
        
        prompt = f"""
你是一位资深的法务专家，精通{jurisdiction}的合同法和相关法规。

请对以下合同进行全面审查：

## 合同内容
{contract_text}

## 审查要求

### 1. 风险识别
请识别以下类型的风险条款：
- 🔴 **高风险**：可能导致重大损失或法律纠纷的条款
- 🟡 **中风险**：可能引发争议或需要明确界定的条款
- 🟢 **低风险**：建议优化的条款

对每个风险条款，请提供：
- 条款位置（章节/条款号）
- 原文引用
- 风险分析（法律依据、潜在问题）
- 修改建议（具体修改方案）

### 2. 合规检查
请检查合同是否符合以下法规：
- 《中华人民共和国合同法》
- 《中华人民共和国民法典》
- 《中华人民共和国个人信息保护法》（如涉及数据处理）
- 《中华人民共和国数据安全法》（如涉及数据安全）
- 行业特殊法规（{industry}）

### 3. 条款完整性
请检查合同是否包含必要条款：
- 合同标的和数量
- 价款或报酬
- 履行期限、地点和方式
- 违约责任
- 争议解决
- 合同生效条件

### 4. 关键条款分析
请提取并分析以下关键条款：
- 付款条款（付款方式、周期、违约金）
- 履约条款（交付时间、验收标准）
- 知识产权条款（权属、授权、侵权）
- 保密条款（保密范围、期限、违约责任）
- 违约责任条款（违约情形、责任承担）

### 5. 审查总结
- 总体风险评级（高/中/低）
- 主要风险点（Top 5）
- 修改优先级建议

## 输出格式
请以结构化的 Markdown 格式输出审查报告，包含：
1. 执行摘要
2. 风险条款列表（按风险等级排序）
3. 合规检查结果
4. 条款完整性检查
5. 关键条款分析
6. 修改建议清单
7. 审查总结

## 注意事项
- 引用具体法律条文
- 提供可操作的修改建议
- 区分强制性和建议性修改
- 考虑商业实践和行业惯例
"""
        
        return prompt
    
    def _parse_review_result(self, result_text: str) -> Dict:
        """解析审查结果"""
        
        # 使用 Claude Code 解析结构化数据
        parse_prompt = f"""
请将以下合同审查报告转换为结构化的 JSON 数据：

{result_text}

输出格式：
{{
  "summary": {{
    "risk_level": "high/medium/low",
    "total_risks": 10,
    "high_risks": 3,
    "medium_risks": 5,
    "low_risks": 2
  }},
  "risks": [
    {{
      "level": "high/medium/low",
      "clause_number": "第X条",
      "original_text": "...",
      "risk_analysis": "...",
      "suggested_revision": "...",
      "legal_basis": "..."
    }}
  ],
  "compliance_issues": [...],
  "missing_clauses": [...],
  "key_clauses": {{...}}
}}
"""
        
        result = self.client.generate(
            prompt=parse_prompt,
            model="claude-3-opus-20240229",
            max_tokens=4000
        )
        
        try:
            return json.loads(result.content)
        except json.JSONDecodeError:
            return {"raw_text": result_text}
    
    def _generate_report(self, review_result: Dict) -> Dict:
        """生成审查报告"""
        
        return {
            "review_summary": review_result.get("summary", {}),
            "risk_analysis": review_result.get("risks", []),
            "compliance_check": review_result.get("compliance_issues", []),
            "completeness_check": review_result.get("missing_clauses", []),
            "key_clauses": review_result.get("key_clauses", {}),
            "recommendations": self._prioritize_recommendations(review_result)
        }
    
    def _prioritize_recommendations(self, review_result: Dict) -> List[Dict]:
        """优先级排序修改建议"""
        
        recommendations = []
        
        # 高风险建议
        for risk in review_result.get("risks", []):
            if risk.get("level") == "high":
                recommendations.append({
                    "priority": "P0",
                    "type": "risk",
                    "description": risk.get("risk_analysis"),
                    "action": risk.get("suggested_revision")
                })
        
        # 合规问题
        for issue in review_result.get("compliance_issues", []):
            recommendations.append({
                "priority": "P1",
                "type": "compliance",
                "description": issue.get("description"),
                "action": issue.get("solution")
            })
        
        return recommendations
```

```python
# src/risk_analyzer.py

from claude_code import Client
import yaml
from typing import List, Dict

class RiskAnalyzer:
    """合同风险分析器"""
    
    def __init__(self, rules_path="config/risk_rules.yaml"):
        with open(rules_path, 'r', encoding='utf-8') as f:
            self.rules = yaml.safe_load(f)
        self.client = Client()
    
    def analyze_risks(
        self,
        contract_text: str,
        risk_types: List[str] = None
    ) -> List[Dict]:
        """
        分析合同风险
        
        Args:
            contract_text: 合同文本
            risk_types: 风险类型列表（financial/legal/operational）
        
        Returns:
            风险列表
        """
        if risk_types is None:
            risk_types = ["financial", "legal", "operational"]
        
        risks = []
        
        for risk_type in risk_types:
            type_risks = self._analyze_risk_type(
                contract_text,
                risk_type
            )
            risks.extend(type_risks)
        
        # 按风险等级排序
        risks.sort(key=lambda x: self._risk_score(x), reverse=True)
        
        return risks
    
    def _analyze_risk_type(
        self,
        contract_text: str,
        risk_type: str
    ) -> List[Dict]:
        """分析特定类型的风险"""
        
        type_config = self.rules["risk_types"][risk_type]
        
        prompt = f"""
请分析以下合同中的{type_config['name']}风险：

## 合同内容
{contract_text}

## 风险类型
{type_config['name']}

## 风险指标
{yaml.dump(type_config['indicators'], allow_unicode=True)}

## 分析要求
1. 识别所有{type_config['name']}相关的风险条款
2. 评估每个风险的可能性和影响程度
3. 提供风险缓解建议

## 输出格式（JSON）
[
  {{
    "risk_type": "{risk_type}",
    "clause": "条款位置",
    "description": "风险描述",
    "likelihood": "high/medium/low",
    "impact": "high/medium/low",
    "mitigation": "缓解措施"
  }}
]
"""
        
        result = self.client.generate(
            prompt=prompt,
            model="claude-3-opus-20240229",
            max_tokens=3000
        )
        
        import json
        try:
            return json.loads(result.content)
        except json.JSONDecodeError:
            return []
    
    def _risk_score(self, risk: Dict) -> int:
        """计算风险分数"""
        likelihood_scores = {"high": 3, "medium": 2, "low": 1}
        impact_scores = {"high": 3, "medium": 2, "low": 1}
        
        likelihood = risk.get("likelihood", "medium")
        impact = risk.get("impact", "medium")
        
        return likelihood_scores.get(likelihood, 2) * impact_scores.get(impact, 2)
```
