### **原則2: Planning before Building**

**重要性**: ⭐⭐⭐⭐⭐

**說明**:
> 先使用 Plan Mode 研究程式碼庫，再執行修改

**實踐**:
```bash

claude --plan

# 2. 研究程式碼庫
# Claude會分析程式碼並提出計畫

# 3. 審核計畫
# 確認計畫合理

# 4. 切換到 Accept Edits

```

**效果**: 返工率降低 **70%**

**重要性**: ⭐⭐⭐⭐

## 🚨 常見陷阱

### **陷阱1: 沒有提供上下文**

**問題**:
```bash
claude generate --prompt "建立 API"
```

**後果**: 生成程式碼不符合預期

**解決方案**:
```bash
claude generate \
  --context "電商平台後端" \
  --prompt "建立使用者 API"
```

---

### **陷阱2: 提示詞過長**

**問題**:
```markdown
我想要你幫我建立一個使用者註冊和登入的功能，
這個功能需要使用 JWT 認證，密碼需要加密儲存，
還要驗證郵箱格式，請幫我寫程式碼。要求程式碼品質高，
有完整的錯誤處理和單元測試。
```

**後果**: Token浪費，效果不好

**解決方案**:
```markdown
建立使用者註冊登入 API：
- JWT 認證
- bcrypt 加密
- 郵箱驗證
```

---

### **陷阱3: 跳過 Plan Mode**

**問題**:
直接修改程式碼

**後果**: 返工率高

**解決方案**:
先用 Plan Mode 研究程式碼庫
