# Claude Code 最佳实践

> 掌握高效使用 Claude Code 的技巧和策略

## 🎯 核心原则

### 1. 清晰明确
- 使用具体、明确的描述
- 提供必要的上下文
- 说明期望的输出格式

### 2. 迭代优化
- 从简单开始，逐步增加复杂度
- 多次迭代改进结果
- 每次只关注一个方面

### 3. 验证测试
- 始终测试生成的代码
- 检查边界条件
- 确保符合项目规范

## 📝 提示词技巧

### ✅ 好的提示词示例

#### 1. 具体明确
```bash
# ❌ 不好
claude-code "写个函数"

# ✅ 好
claude-code "创建一个 Python 函数 calculate_bmi，接受体重（kg）和身高（m）作为参数，返回 BMI 值和健康状态（偏瘦/正常/超重）"
```

#### 2. 提供上下文
```bash
# ❌ 不好
claude-code "创建一个 API 端点"

# ✅ 好
claude-code "在现有的 Flask 应用中添加一个 POST 端点 /api/users/register，用于用户注册。
需要验证：
1. 邮箱格式
2. 密码强度（至少 8 位，包含数字和字母）
3. 用户名唯一性
返回 JSON 格式的成功/失败消息"
```

#### 3. 指定技术栈
```bash
# ❌ 不好
claude-code "连接数据库"

# ✅ 好
claude-code "使用 SQLAlchemy 创建一个数据库连接，连接到 PostgreSQL 数据库。
配置信息：
- 数据库：myapp
- 用户：admin
- 密码从环境变量 DB_PASSWORD 读取
使用连接池，池大小为 5"
```

#### 4. 包含示例
```bash
# ✅ 好
claude-code "创建一个 JavaScript 函数，行为类似于这个 Python 函数：

def format_currency(amount, currency='USD'):
    symbols = {'USD': '$', 'EUR': '€', 'GBP': '£'}
    symbol = symbols.get(currency, currency)
    return f'{symbol}{amount:.2f}'

但用 TypeScript 编写，并添加类型注解"
```

## 🔄 工作流程最佳实践

### 1. 分阶段开发

#### 阶段 1：需求分析
```bash
# 先描述需求和设计
claude-code "设计一个用户认证系统的架构，包括：
1. 注册流程
2. 登录流程
3. 密码重置
4. 会话管理
请提供架构图和关键组件说明"
```

#### 阶段 2：核心功能
```bash
# 实现核心功能
claude-code "基于上面的设计，实现用户注册功能，包括：
1. 输入验证
2. 密码加密
3. 数据库存储
4. 邮件验证"
```

#### 阶段 3：测试
```bash
# 生成测试
claude-code "为用户注册功能生成完整的测试用例，包括：
1. 正常情况
2. 边界条件
3. 异常处理"
```

### 2. 代码审查循环

```bash
# 步骤 1：生成代码
claude-code "创建一个数据验证函数"

# 步骤 2：审查代码
claude-code --file validator.py "审查这个文件，提出改进建议"

# 步骤 3：应用改进
claude-code --file validator.py "根据审查建议优化代码"

# 步骤 4：生成文档
claude-code --file validator.py "为这个文件生成详细的文档"
```

### 3. 测试驱动开发（TDD）

```bash
# 步骤 1：编写测试
claude-code "为用户登录功能编写 pytest 测试用例：
1. 正确的用户名和密码
2. 错误的密码
3. 不存在的用户
4. 账户被锁定"

# 步骤 2：实现功能
claude-code "根据测试用例实现用户登录功能，确保所有测试通过"

# 步骤 3：运行测试
pytest tests/test_login.py -v
```

## ⚡ 性能优化最佳实践

### 1. 选择合适的模型

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 代码生成 | Opus | 质量最高，复杂逻辑 |
| 代码审查 | Sonnet | 平衡质量和速度 |
| 简单查询 | Haiku | 快速响应，低成本 |
| 文档生成 | Sonnet | 足够好，性价比高 |

### 2. 优化上下文

```bash
# ❌ 不好 - 包含整个项目
claude-code --dir . "分析整个项目"

# ✅ 好 - 只包含相关文件
claude-code --file src/auth/login.py "优化这个函数"
```

### 3. 批量处理

```bash
# ❌ 不好 - 多次调用
claude-code "创建文件 A"
claude-code "创建文件 B"
claude-code "创建文件 C"

# ✅ 好 - 一次调用
claude-code "创建以下文件：
1. src/models/user.py - 用户模型
2. src/models/product.py - 产品模型
3. src/models/order.py - 订单模型"
```

## 🔒 安全最佳实践

### 1. 不要提交敏感信息

```bash
# ❌ 危险
claude-code "连接到数据库，密码是 'my-secret-password'"

# ✅ 安全
claude-code "连接到数据库，密码从环境变量读取"
```

### 2. 审查生成的代码

```bash
# 始终检查安全问题
claude-code --file generated_code.py "审查这段代码的安全问题：
1. SQL 注入
2. XSS 攻击
3. CSRF 漏洞
4. 敏感信息泄露"
```

### 3. 使用安全库

```bash
claude-code "使用 bcrypt 库实现密码哈希，不要自己实现加密算法"
```

## 🚀 高级技巧

### 1. 链式命令


### 2. 使用模板


## 📈 效率提升技巧

### 1. 快捷键和别名

```bash
# 在 .bashrc 或 .zshrc 中添加别名
alias cc="claude-code"
alias ccf="claude-code --file"
alias ccr="claude-code --recursive"

# 使用
cc "创建一个函数"
ccf script.py "优化这个文件"
```

### 2. 批处理脚本

```bash
#!/bin/bash
# batch-review.sh

for file in src/**/*.py; do
  echo "Reviewing $file..."
  claude-code --file "$file" "审查这个文件"
done
```

### 3. 集成到 CI/CD

```yaml
# .github/workflows/claude-code-review.yml
name: Claude Code Review
on: [pull_request]
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Review Code
        run: |
          claude-code --dir . "审查修改的代码" > review.md
      - name: Comment PR
        uses: actions/github-script@v5
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: review
            });
```
